# 1. 整体项目说明

## 1.1 项目定位

AI 短剧生产框架是一套面向连续剧式内容生产的 AI 原生生产操作系统。它管理从故事策划、剧本结构化、分镜、角色与场景资产、视频与声音生成，到后期合成、质量检测、人工审核和发布导出的完整链路。

系统的核心不是某个具体生成模型，而是以下四项长期资产：

1. **Canonical Schema**：模型无关的权威叙事与镜头数据结构。
2. **Compilation Pipeline**：从事实数据编译到模型请求的确定性流程。
3. **Workflow Orchestrator**：可恢复、可重试、可人工接管的生产 DAG。
4. **Evaluation System**：把模型输出转化为可测量、可审核、可迭代的生产结果。

## 1.2 要解决的问题

直接使用提示词生成短剧，会快速遇到以下问题：

- 角色、服装、场景和道具在镜头间漂移；
- 无法追踪某个镜头使用了哪版角色、提示词和参考素材；
- 修改世界观或服装后，不能自动识别受影响镜头；
- 不同模型使用不同请求结构，导致业务数据被供应商绑定；
- 台词、音色、口型、环境声和镜头时长互相耦合；
- 单个镜头可以“很好看”，但整集叙事不连贯；
- 模型失败后只能整段重做，不能局部恢复；
- 生成结果会被误认为事实，污染后续连续性。

框架通过结构化事实源、依赖关系、版本血缘和审核状态解决这些问题。

## 1.3 核心设计原则

### 原则 A：结构化事实源优先

所有长期有效的信息必须存在于结构化领域模型中，包括：

- 人物身份和版本；
- 服装、妆造和伤病状态；
- 场景、时间、天气和光照；
- 道具归属、位置和状态；
- 剧情因果、人物动机和关系；
- 镜头开始/结束状态；
- 导演意图与声音策略。

提示词只是一种编译产物。

### 原则 B：事实、导演表达、模型执行分层

系统中存在三种表示：

1. **语义事实层**：故事世界中真实发生了什么。
2. **导演表达层**：观众应该如何看到和听到这些事实。
3. **模型执行层**：某个具体供应商所需的提示词、参数和参考素材。

更换模型时，前两层原则上不变。

### 原则 C：生成结果不得自动回写事实源

每个镜头都区分：

- `planned_state`：计划状态；
- `detected_state`：从生成结果检测到的状态；
- `approved_state`：人工或规则最终批准的状态。

模型生成“左手拿杯子”不能自动覆盖计划中的“右手拿杯子”。系统应报告差异，并由审核流程决定重生成、接受偏差或修改后续状态。

### 原则 D：架构完整不等于首版全部自动化

每个能力必须拥有稳定接口，但实现可以是：

- `AUTOMATED`：自动实现；
- `RULE_BASED`：规则或工具实现；
- `MANUAL`：人工上传或人工确认；
- `UNAVAILABLE`：明确阻塞，不伪造成功结果。

这样可以在不重构领域层和 DAG 的前提下逐步替换实现。

### 原则 E：一切生成必须可追溯

每个资产记录：

- 来源模型与版本；
- Provider；
- 请求参数与编译后提示词；
- 参考资产及其版本；
- 随机种子（若可用）；
- 生成任务、耗时、错误与重试；
- 审核结果和被哪些成片版本采用。

## 1.4 产品边界

### MVP 范围内

- 项目、剧集圣经、角色、场景和道具管理；
- Episode → Scene → Beat → Shot 结构化编剧；
- 自动拆镜头和导演意图生成；
- Seedream 分镜图、角色图、关键帧；
- Seedance 视频和原生音频；
- GPT-SoVITS 权威配音；
- Suno 音乐接入；
- Seedance 派生音效；
- 时间线、字幕、混音和 FFmpeg 导出；
- Gemini 原生视频理解；
- CV 逐帧分析和媒体技术检测；
- Opus/Sol 叙事审核；
- 人工审核、版本回退和局部重生成；
- 完整 Provider 与 Stub 架构。

### 不作为首版硬要求

- 全自动取代人工导演；
- 每个镜头强制口型同步；
- 每句对白强制逐字对齐；
- 多供应商视频模型自动竞价；
- 完整专业 NLE 的所有高级功能；
- 无人工审核的一键自动发布。

## 1.5 典型用户

- 短剧策划与编剧；
- AI 导演与分镜师；
- 角色/场景视觉设计师；
- 视频生成与后期人员；
- 声音设计和配音人员；
- 项目负责人和审核人员；
- 负责接入模型、算力和内部工具的工程师。

## 1.6 核心成功指标

- 有效镜头采用率；
- 角色身份一致性；
- 服装、道具和空间连续性；
- 台词准确率与音色一致性；
- 单镜头平均重试次数；
- `Generation Multiplier = 总生成秒数 / 最终采用秒数`；
- 单次修改影响范围是否可被准确识别；
- 失败任务能否从检查点恢复；
- 人工审核平均处理时长；
- 一集从结构化剧本到可导出成片的闭环完成率。
