# 4. 模型与技术选型

## 4.1 选型原则

- 仅采用当前已具备访问条件的主要生成模型；
- 模型按职责分工，不追求单模型端到端；
- 每个模型都通过 Provider Adapter 接入；
- 模型版本、请求和结果必须可追溯；
- 允许原生音视频、权威 TTS 和混合音频三种路线共存；
- 专用修复能力暂时可为 Stub，但接口必须存在。

## 4.2 文本与推理模型

### Claude Opus 4.8

- 主要职责：世界观、整季主线、人物弧、关键创作、叙事终审。
- 不负责：高频 Schema 编译和批量标签。

### Claude Sonnet 5

- 主要职责：单集扩写、对白、批量改写、日常编剧。
- 不负责：最终事实校验。

### GPT-5.6 Sol

- 主要职责：复杂逻辑、跨模型仲裁、质量报告归因。
- 不负责：所有轻量任务。

### GPT-5.6 Terra

- 主要职责：剧本结构化、镜头拆解、状态迁移和 Generation IR。
- 不负责：最高自由度文学创作。

### GPT-5.6 Luna

- 主要职责：分类、枚举归一、引用检查、Prompt Lint 和轻量 QC。
- 不负责：长篇创作和复杂仲裁。

### Gemini

- 主要职责：直接读取视频/音频，生成事件时间线和语义报告。
- 不负责：帧级精确测量和最终叙事裁决。

### 推荐协作模式

```text
Opus 创作
→ Sonnet 扩写
→ Terra 编译
→ Luna 校验
→ Seedream / Seedance 生成
→ Gemini 理解
→ CV 测量
→ Opus / Sol 仲裁
```

## 4.3 图像：Seedream 5.0

承担：

- 角色定妆和多视图；
- 表情板、服装版本；
- 场景和道具母版；
- 分镜图；
- 视频首帧、尾帧和关键帧；
- 局部重绘和候选变体。

必须锁定经审核的角色、服装和场景版本，并在 Shot 中使用版本 ID 引用。

## 4.4 视频与原生音频：Seedance 2.0

承担：

- 文生/图生视频；
- 多素材参考；
- 参考音频或视频驱动；
- 原生对白、环境声和动作音效；
- 动作、转场和主要镜头；
- 派生音效源视频。

Seedance 作为第一版唯一主要视频 Provider 足够，但业务层不得出现 Seedance 专用字段。所有请求先进入 `VideoGenerationSpec`。

## 4.5 配音：GPT-SoVITS

承担：

- 主角固定音色；
- 权威台词；
- 中文对白与旁白；
- 角色声音版本；
- 情绪参考音频驱动。

框架需要补足：

- 参考音频管理；
- 音色授权记录；
- 分句生成；
- 停顿与语速策略；
- 后处理和响度标准化；
- 句级字幕时间推导。

## 4.6 音乐：Suno Provider

Suno 仅作为 `MusicGenerationProvider` 的一个实现。领域数据保存音乐意图、段落、情绪曲线和使用位置，不保存 Suno 专属字段。

必须保留：

- `ManualMusicImportProvider`；
- Provider 替换能力；
- 原始生成记录和授权元数据。

## 4.7 声音策略

### Seedance Native Audio

适合短对白、环境声、动作音画强耦合的镜头。采用条件：

- 台词与权威剧本一致或被明确批准；
- 角色音色可接受；
- 音质、口型和环境声通过 QC；
- 后续不要求独立多语言替换。

### GPT-SoVITS First

```text
权威台词
→ GPT-SoVITS
→ 最终对白音频
→ 作为 Seedance 参考输入
→ 视频生成
```

适合主角和关键对白。

### Hybrid

Seedance 提供环境声/动作声，GPT-SoVITS 提供权威对白，后期混合。若 Seedance 输出混合轨，需要可选音源分离或重新生成更干净的声音层。

## 4.8 生成式音效：Seedance Derived SFX

```text
SoundEffectSpec
→ Seedance 生成单一声音事件视频
→ FFmpeg 提取音轨
→ 截取/去静音/降噪/响度标准化
→ 审核
→ 写入内部 SFX 素材库
```

适合爆炸、关门、脚步、机械、魔法、雷雨、破碎等事件。

内部素材库检索优先于重新生成：

```text
查找已批准素材
→ 命中则复用
→ 未命中才生成
→ 新素材审核后回填
```

## 4.9 后期技术栈

- FFmpeg / ffprobe：转码、拼接、抽取、封装、字幕、混音、响度；
- PostgreSQL：时间线元数据与版本；
- MinIO/S3：原始素材、代理文件和导出物；
- Vue 时间线：非破坏性编辑；
- Python Media Worker：波形、缩略图、代理、音视频处理。

## 4.10 质检技术栈

### 原生视频理解

Gemini 直接分析视频和音频，输出：

- 事件时间线；
- 人物动作、情绪和对白；
- 镜头语义；
- 是否完成导演意图；
- 可疑时间区间。

### CV

- OpenCV：光流、冻结帧、重复帧、闪烁、亮度变化；
- MediaPipe：人脸和姿态关键点；
- 身份嵌入：角色一致性；
- 可插拔对象检测/跟踪：道具与角色数量；
- 可选 SyncNet 风格评分：口型同步。

### 叙事终审

Opus 或 Sol 接收：

- 权威 ShotExpectation；
- Gemini 报告；
- CV 报告；
- 前后镜头上下文；

输出 `pass / warning / manual_review / repair / regenerate`。

## 4.11 当前保留 Stub 的能力

- 专用口型修复；
- 高精度强制对齐；
- 专用纯音效生成；
- 高级音源分离；
- 第二视频 Provider；
- 专用真人表演迁移。

这些 Stub 不阻塞主链路，但必须产生明确的 `manual_action_required` 或降级策略。
