# 6. 后期质检与审核体系

## 6.1 为什么需要多层质检

通用多模态模型擅长“理解”，CV 擅长“测量”，FFmpeg/ffprobe 擅长“确定性技术检查”，Opus/Sol 擅长“判断这些问题在叙事上意味着什么”。任何单一层都不能替代其他层。

## 6.2 第一层：媒体技术 QC

使用 FFmpeg、ffprobe 和规则检测：

- 文件可解码；
- 时长、分辨率、帧率和纵横比；
- 音轨是否存在；
- 黑帧、静音和截断；
- 峰值、响度和削波；
- 重复帧和冻结；
- 字幕越界；
- 导出规格是否匹配平台。

此层结果是确定性指标，不使用 LLM。

## 6.3 第二层：Gemini 原生视频理解

Gemini 直接接收视频/音频，输出：

- 时间标记的事件；
- 角色动作；
- 情绪与表演；
- 谁在说话、说了什么；
- 镜头运动与构图描述；
- 与导演意图的语义偏差；
- 值得进一步逐帧检查的时间区间。

示例：

```json
{
  "events": [
    {
      "start_ms": 2100,
      "end_ms": 3900,
      "actor": "char_lin",
      "action": "pick_up_cup",
      "hand": "left",
      "emotion": "hesitant"
    }
  ],
  "dialogue_match": "semantic_match",
  "camera": "mostly_static"
}
```

Gemini 的时间判断属于语义证据，不作为帧级真值。

## 6.4 第三层：CV 逐帧分析

### 画面稳定性

- 光流异常；
- 闪烁；
- 亮度/颜色突变；
- 重复帧和冻结；
- 局部纹理跳变。

### 人物身份

- 人脸检测和跟踪；
- 身份嵌入相似度曲线；
- 中途变脸时间区间；
- 多余或缺失人物。

### 姿态和运动

- 人体关键点；
- 肢体轨迹连续性；
- 瞬移、跳变和不自然速度；
- 镜头推进/拉远的数值估计。

### 道具与场景

- 指定道具是否出现；
- 道具消失和位置突变；
- 服装主色或纹理明显变化；
- 角色数量和空间位置。

### 音画

- 嘴部运动与音频活动区间；
- 音频领先/滞后；
- 说话者匹配；
- 对白区间内嘴部是否静止。

## 6.5 第四层：叙事审核

Opus 或 Sol 接收：

- Shot 的叙事目的；
- planned/approved continuity state；
- Gemini 语义报告；
- CV 帧级指标；
- 相邻镜头；
- 台词与人物设定。

输出：

```json
{
  "decision": "regenerate",
  "severity": "high",
  "reasons": [
    "情绪呈现为犹豫，不符合克制愤怒",
    "机位近乎静止，未形成计划中的压迫感",
    "左手拿杯子破坏上一镜头连续性"
  ],
  "recommended_scope": "entire_shot",
  "suggested_changes": [
    "提高表演强度",
    "明确 slow push-in",
    "强化右手拿杯子的动作约束"
  ]
}
```

## 6.6 质量决策引擎

统一决策：

- `pass`
- `pass_with_warning`
- `manual_review`
- `repair`
- `regenerate`
- `blocked`

规则优先于 LLM：文件损坏、缺失音轨等硬错误必须直接失败。

## 6.7 人工审核工作台

审核者应能：

- A/B/C 候选同步播放；
- 查看 expected vs detected 差异；
- 跳转异常帧或时间区间；
- 接受、拒绝、局部修复、重新生成；
- 修改事实源或仅接受当前偏差；
- 锁定角色、服装、关键帧和音频；
- 将修改意见回送给 Prompt Compiler。

## 6.8 质量阈值

阈值按项目和镜头类型配置。示例：

- 主角特写身份相似度要求高于环境远景；
- 口型只在嘴部可见且有对白时启用；
- 动作镜头允许更高的纹理波动；
- 道具特写必须严格检查目标道具；
- 原生 Seedance 音频必须进行台词比对和音色审核。

## 6.9 模型输出不自动成为事实

任何 `detected_state` 只能产生报告。只有以下操作可生成 `approved_state`：

- 人工批准；
- 明确配置的确定性规则；
- 可审计的批量批准策略。
