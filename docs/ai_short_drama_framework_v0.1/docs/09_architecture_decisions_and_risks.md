# 9. 架构决策与风险

## ADR-001：结构化数据是唯一事实源

**决定：** 提示词、模型请求和生成资产均为派生数据。  
**原因：** 保持可替换性、连续性和可追溯性。  
**影响：** 需要投入设计 Canonical Schema 和编译器。

## ADR-002：事实层、导演层、执行层分离

**决定：** 同一剧情事实可拥有多个导演方案和多个模型执行方案。  
**影响：** 支持候选镜头、不同平台改编和模型切换。

## ADR-003：生成结果不自动覆盖事实

**决定：** 使用 planned/detected/approved 三态。  
**影响：** 系统必须提供差异审核和状态批准。

## ADR-004：架构完整、实现可降级

**决定：** 首版保留全部能力接口，未实现部分使用 Manual/Unavailable Stub。  
**影响：** 不因缺少口型、对齐或第二视频模型而改变 DAG。

## ADR-005：Gemini 负责原生视频理解

**决定：** Gemini 直接分析视频和音频；Opus/Sol 负责叙事裁决。  
**原因：** 原生感知与高层判断职责不同。

## ADR-006：CV 与原生视频理解并存

**决定：** Gemini 负责语义，CV 负责帧级测量。  
**原因：** 原生视频模型可能漏掉短暂异常，CV 不理解剧情。

## ADR-007：Seedance 原生音频是一等公民

**决定：** 原生音频可直接批准进入成片，不强制替换。  
**条件：** 台词、音色、音质、口型和授权通过审核。

## ADR-008：支持三种声音策略

- Seedance Native；
- GPT-SoVITS First；
- Hybrid。

系统按镜头选择，不设全局唯一策略。

## ADR-009：Seedance 派生音效进入内部素材库

**决定：** 可生成视觉事件并提取音频作为 SFX。  
**影响：** 需要音频提取、后处理、质量审核和授权元数据。

## ADR-010：口型与对齐是条件能力

**决定：** 接口存在，但不是所有 Shot 必跑。  
**原因：** 原生音视频或声音先行镜头可能天然同步；句级字幕可由时间线推导。

# 风险登记

## 模型和 API 变化

**风险：** 参数、能力、版本和服务状态变化。  
**控制：** Provider Adapter、能力协商、版本快照、基准测试。

## 角色身份漂移

**风险：** 多镜头角色脸型、服装和身材变化。  
**控制：** 角色版本锁定、参考资产、CV 身份曲线、人工审核。

## 原生音频不可拆分

**风险：** 对白、环境声和音乐混在单轨。  
**控制：** 重要对白采用 GPT-SoVITS First；Hybrid 需要分离或重生成干净轨。

## Seedance 派生音效不干净

**风险：** 包含人声、音乐或额外环境声。  
**控制：** 单事件提示、抽取后处理、质量门、失败时人工上传。

## LLM 生成无效 Schema

**风险：** 引用不存在、状态矛盾。  
**控制：** Terra 结构化输出 + 确定性验证器 + Luna lint；不直接写数据库。

## 过早复杂化领域模型

**风险：** Schema 过细导致创作困难。  
**控制：** 核心字段稳定，扩展字段使用 typed extension；通过真实短剧基准迭代。

## CV 误报

**风险：** 艺术性镜头被误判为闪烁或身份变化。  
**控制：** 按镜头类型配置阈值；CV 不直接作叙事决定。

## 工作流重入与重复资产

**风险：** 回调重复或任务重试产生多份输出。  
**控制：** 幂等键、内容哈希、单向状态机和去重。

## 授权与合规

**风险：** 声音克隆、人物肖像、音乐和素材授权不清。  
**控制：** Asset License Metadata、授权文件、角色/音色使用范围和审计日志。

## 数据与密钥安全

**风险：** API Key、未发布剧本和角色资产泄露。  
**控制：** 密钥管理、最小权限、签名 URL、日志脱敏、隔离开发和生产环境。
